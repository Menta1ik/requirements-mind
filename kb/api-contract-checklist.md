# Чек-лист: API Contract (Спецификация API проекта)

Этот чек-лист используется агентом **A2 Validator** для проверки полноты спецификаций API и агентом **A4 Writer** в качестве шаблона для генерации документа.

---

## 1. Базовая информация об API
- [ ] Указан базовый URL и поддерживаемые версии API (версионирование через URL или заголовки).
- [ ] Описаны используемые форматы данных (JSON, XML, Protocol Buffers).
- [ ] Задокументированы общие HTTP-заголовки запросов и ответов (например, `Content-Type`, `Authorization`).

---

## 2. Спецификация эндпоинтов (Endpoints)
- [ ] Для каждого эндпоинта указан HTTP-метод (`GET`, `POST`, `PUT`, `DELETE`, `PATCH`).
- [ ] Описаны все query-параметры, path-параметры и их типы данных.
- [ ] Приведен полный пример тела запроса (Request Body) с описанием обязательности полей.
- [ ] Описана валидация входных данных (максимальная длина, форматы строк, диапазоны чисел).

---

## 3. Модели ответов и коды статусов
- [ ] Приведены детальные примеры успешных ответов (например, HTTP 200 OK, 201 Created).
- [ ] Задокументированы примеры всех возможных ошибок (например, HTTP 400 Bad Request, 401 Unauthorized, 404 Not Found, 422 Unprocessable Entity, 500 Internal Server Error).
- [ ] Ошибки оформлены строго по стандарту RFC 7807 (Problem Details).

---

## 4. Безопасность и ограничения
- [ ] Указана схема авторизации для каждого эндпоинта (требуется ли авторизация, какой тип токена).
- [ ] Документирована политика ограничения частоты запросов (Rate Limiting) для эндпоинтов.

---

## 5. Интерактивные примеры
- [ ] Приведены рабочие примеры вызовов через `curl` или спецификацию в формате OpenAPI / Swagger 3.0.
