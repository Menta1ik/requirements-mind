# Glossary / Глоссарий

Общий глоссарий проекта. Пополняется по ходу работы над документами.

## Термины

| Термин | Определение | Источник |
|---|---|---|
| BRD | Business Requirements Document — документ бизнес-требований | BABOK v3 |
| SRS | Software Requirements Specification — спецификация требований к ПО | ISO 29148 |
| Tech Design | Technical Design Document — документ технического дизайна | — |
| API Contract | Спецификация программного интерфейса (OpenAPI / AsyncAPI) | — |
| FR | Functional Requirement — функциональное требование | ISO 29148 |
| NFR | Non-Functional Requirement — нефункциональное требование | ISO 29148 |
| BR | Business Rule — бизнес-правило | BABOK v3 |
| MoSCoW | Must Have / Should Have / Could Have / Won't Have — метод приоритизации | BABOK v3 |
| RACI | Responsible / Accountable / Consulted / Informed — матрица ответственности | PMBOK 7 |
| AS-IS | Текущее состояние процесса или системы | — |
| TO-BE | Желаемое будущее состояние | — |
| Happy Path | Основной сценарий без ошибок и исключений | IEEE 830 |
| Traceability | Трассируемость — связь между требованиями разных уровней | ISO 29148 |
| PO | Product Owner — владелец продукта | Scrum Guide |
| BA | Business Analyst — бизнес-аналитик | BABOK v3 |
| SLA | Service Level Agreement — соглашение об уровне сервиса | — |
| MTTR | Mean Time To Recovery — среднее время восстановления | — |
| MTBF | Mean Time Between Failures — среднее время между сбоями | — |
